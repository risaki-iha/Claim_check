# 解約リスク検知くん リアルタイム版

Slackの社内/社外チャンネルを分析して解約・契約見直し・クライアント不満の兆候をスレッドの文脈込みで検知する。

## チャンネルID（固定）
- `通知先（即送信）`: C0AHU6M9GC8

## 検知対象スコープ
- **対象**: チャンネル名に `社内` または `社外` が含まれるチャンネルのみ
- **除外**: `mdx_`、`dxm_` 等の部署内チャンネル、チャンネル名に「hajimari」を含むもの
- **除外**: Botメッセージ
- **クレーム系は対象外**: クレーム・ミス・対応遅れ等は「クレーム検知くん」の管轄

---

## 重要度マトリクス

### 🔴 即対応・上長報告
- 解約・契約終了・更新しないを明示
- 内製化・発注終了など具体的時期あり
- 即日対応必要

### 🟡 要対応・要確認
- 明確な不満・改善要求
- 他社サービスとの比較・検討開始
- 来期方針について継続判断中
- 担当者変更の話題

### 🔵 情報共有
- 予算・コスト「削減」の話が出始めた、または継続予算に慎重な含みが見える段階（前向き・中立な費用検討＝新規提案・定例の費用すり合わせは🔵ではなくNOISE）
- クライアントのトーンが慎重・遠慮がちになっている（初期の予兆段階）
- 解決方向が見えているが記録として残したいもの

### NOISE（除外）
- 短すぎるメッセージ、主語不明
- 社内承認用途のやり取り
- ポジティブな文脈（「予算上振れ」「来期も継続」など解約と逆の話題）
- クレーム系（ミス・対応遅れ等は「クレーム検知くん」の管轄なのでこちらでは検知しない）
- **通常の商談・MTG議題**: ロードマップ・費用感のすり合わせ、提案資料の確認、定例打ち合わせ、見積り提示など、**特に解約リスク示唆や予算削減示唆のない業務的議論はNOISE**。「費用」「予算」というワードが出ても、新規提案・既存施策の費用見直しなど前向き/中立な文脈は除外。
- **検討段階の議論**: 単に「予算を確認する」「費用感を共有する」など、確定的な削減・撤退の意向が無い段階の話題はNOISE。

## チャンネル×投稿者の判定軸

| チャンネル | 投稿者 | 検知基準 |
|-----------|-------|---------|
| 社外_ | クライアント | 解約示唆・予算削減・契約見直しの直接発言 |
| 社外_ | 社内スタッフ | クライアント側の解約リスク発言を共有・対応相談 |
| 社内_ | 社内スタッフ | 取引縮小・予算削減の社内共有、来期方針の社内議論 |

## 出力ガイドライン

各スレッドについて以下の構造で判定する:

```json
{
  "channel_name": "社外_クライアント名",
  "channel_id": "C0ABC123",
  "thread_ts": "1234567890.000000",
  "permalink": "https://...",
  "importance": "🔴 即対応・上長報告",
  "summary": "クライアントから来期の予算削減方針が明示された。継続判断は来月の定例で決定予定。",
  "main_owner_name": "山田太郎",
  "main_owner_email": "yamada@nyle.co.jp",
  "thread_summary": "..."
}
```

- `summary`: スレッドの文脈を踏まえた1〜2行の要約（リスクの内容と緊急度がわかる形で）
- `main_owner_name`: スレッドにメインで対応している社内スタッフ（@nyle.co.jp）の表示名
- `main_owner_email`: 同スタッフのメール

## 通知本文の組み立て（参考）

Slack 通知本文は Python 側の `build_notification_text` が組み立てる。AI の応答は本書の出力スキーマに従う JSON 配列のみで良い。

通知ブロックには 🔴 🟡 の場合のみ【マネージャー】行が追加される:
- マスタスプシ `1PWqW08yD6shJu5sRUxTZvf7w7K7TaJEuQcDU2QmkZXY` の「【公開用】案件ごとAM一覧」シートを参照（D列＝マネ名 / E列＝メアド / G列＝slackチャンネルID）
- マッチング順序: ①G列(slackチャンネルID)完全マッチ → ②B列(コール名)でチャンネル名部分マッチ
- 解決後のメンション化は E列メアド → Slack user_id を優先、引けなければ D列マネ名 → Slack user_id をフォールバック
- どちらも引けなければ【マネージャー】行は出さない（事故メンション防止）
- 🔵 は出さない
- テスト時はマスタスプシのD列/E列を直接書き換えて運用する（環境変数による差し替えは行わない）

## 実行時の注意

- 解約リスク以外のテーマ（クレーム・ミス・対応遅れ等）は NOISE として除外する
- 「解約」というキーワードがヒットしても、文脈が「解約防止策の議論」「過去事例の共有」などリスク以外なら NOISE
- 同じスレッドで複数の発言が解約リスク示唆している場合は、最も重い発言の重要度を採用
